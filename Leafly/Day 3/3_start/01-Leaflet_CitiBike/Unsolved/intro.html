Basic Version

Take a look at this json file: https://gbfs.citibikenyc.com/gbfs/en/station_information.json

Notice we have latitude and longitude, but it is not geojson data. It is just plain .json

Notice that the response also includes other useful information, such as the `name`, `station_id`, and `capacity` of each bike station.

Advanced Version

This version groups markers into layers according to station status, e.g. "Out of Order", "Coming Soon", or "Empty".

Using a Leaflet plugin, different custom markers are used for each status group.

See this leaflet plugin: https://github.com/coryasilva/Leaflet.ExtraMarkers

Similar to the first example, when a marker is clicked, a popup appears. Additionally, we see the number of bikes available at each station along with the station name and total capacity.

This version of the activity uses a map legend to display quantities of markers of each status, as well as their colors and the last time the API was updated.

You'll also need the Citi Bike API Station Status Endpoint: https://gbfs.citibikenyc.com/gbfs/en/station_status.json.
  • This endpoint provides information about the status of each bike station, including the number of bikes available and whether or not the station is currently installed and renting bikes.
  • This endpoint does not, however, provide coordinates for each bike station. This must still be obtained from the first endpoint we discussed, the Citi Bike API Station Information Endpoint https://gbfs.citibikenyc.com/gbfs/en/station_information.json.
  • The stations in both API responses have corresponding `station_id` properties. This should make it easier to match up and utilize data in both datasets.


Complete the basic version of the activity before moving on to this more advanced version.
